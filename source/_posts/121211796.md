比如使用`sudo apt install deepin-wine-helper`安装的时候


> 
> 下列软件包有未满足的依赖关系：
 libgirepository-1.0-1 : 破坏: python-gi (< 3.34.0-4~) 但是 3.30.4-1 正要被安装

使用`apt show  libgirepository-1.0-1 -a`可以看到里面的“Breaks:”行

```bash
Package: libgirepository-1.0-1
Version: 1.66.1-1+b1
Priority: optional
Section: libs
Source: gobject-introspection (1.66.1-1)
Maintainer: Debian GNOME Maintainers <pkg-gnome-maintainers@lists.alioth.debian.org>
Installed-Size: 265 kB
Depends: libglib2.0-0 (>= 2.62.4-2~), libc6 (>= 2.29), libffi7 (>= 3.3~20180313)
Breaks: libcjs0 (<< 4.4.0-2~), libgjs0g (<< 1.58.1-2~), libglib-object-introspection-perl (<< 0.048-2~), python-gi (<< 3.34.0-4~), python3-gi (<< 3.34.0-4~), ruby-gobject-introspection (<< 3.4.1-2~)
Homepage: https://wiki.gnome.org/GObjectIntrospection
Tag: role::shared-lib
Download-Size: 96.7 kB
APT-Manual-Installed: yes
APT-Sources: https://mirrors.tuna.tsinghua.edu.cn/debian bullseye/main amd64 Packages

```
意思就是，libgirepository-1.0-1与版本小于3.34.0-4的python-gi冲突。

我们安装的包deepin-wine-helper版本为3.30.4-1，恰好符合冲突范围。
也可以看到python-gi依赖于这个包，
![在这里插入图片描述](https://cdn.jsdelivr.net/gh/fjh1997/CSDN@main/source/images/606be15677817e5c5192aefac60bef66.png)

所以正确的方法是调整deepin-wine-helper包依赖的版本。
那么如何调整？

使用

```bash
sudo apt-cache depends deepin-wine-helper --recurse> depends.log

```
查看这个包的依赖情况。打印的日志大概是先序遍历的。

找到python-gi：
![在这里插入图片描述](https://cdn.jsdelivr.net/gh/fjh1997/CSDN@main/source/images/b3aea2f6563cba6bd80016084e420fc7.png)
看到是python-gobject依赖python-gi

![在这里插入图片描述](https://cdn.jsdelivr.net/gh/fjh1997/CSDN@main/source/images/e150d6affa41d64b8a3d5e41a90542cd.png)

deepin-wine-plugin依赖python-gobject
查了一下，deepin-wine-plugin依赖的python-gobject没有限定版本。
![在这里插入图片描述](https://cdn.jsdelivr.net/gh/fjh1997/CSDN@main/source/images/32fd9cde1e1cea03068b12514254d56a.png)


同样的，python-gobject也没有限定python-gi的版本
![在这里插入图片描述](https://cdn.jsdelivr.net/gh/fjh1997/CSDN@main/source/images/551e0b31001373f8975fa78e5c283b74.png)
那么我们只要安装一个新版本python-gi即可解决问题。

但是看了一下python-gi，发现默认的源只有3.30的版本，新版本要手动编译。
由于python-gi是python-gobject的一个子集，所以我们下载pygobject的源码编译即可。
编译需要源码，源码去哪找？
你们有没有发现source.list里面的deb-src被注释了，其实这个就是源码，取消注释之后apt-update可以去看看。

```bash
 apt-get  source pygobject 
```
![在这里插入图片描述](https://cdn.jsdelivr.net/gh/fjh1997/CSDN@main/source/images/2974a9fe3555bb56d512775a88060f7e.png)

发现来自https://salsa.debian.org/gnome-team/pygobject，去看看
发现含有python-gi.install的是最后一个版本是3.36.0-3：

那么我们clone源码就可以了

```bash
sudo apt install dpkg-dev devscripts
git clone -b debian/3.36.0-3 https://salsa.debian.org/gnome-team/pygobject.git
git clone -b debian/1.16.2-3 https://salsa.debian.org/python-team/packages/pycairo.git
sed -i "s/python-dbg/python2-dbg/g" pycairo/debian/control
#这里python2-dbg实际发现和python-dbg其实是同一个包，只不过在debian bullseye 里替换成了python2-dbg了。

sudo mk-build-deps --install pycairo/debian/control
cd pycairo
sudo debuild -b -uc -us
sudo dpkg -i ../python-cairo_1.16.2-3_amd64.deb
sudo dpkg -i  ../python-cairo-dbg_1.16.2-3_amd64.deb
sudo dpkg -i ../python-cairo-dev_1.16.2-3_all.deb

cd ..
#解决依赖问题
sudo mk-build-deps --install pygobject/debian/control.in
cd pygobject
sudo debuild -b -uc -us
dpkg -i ../python-gi_3.36.0-3_amd64.deb

```
之后就安装成功，撒花！

同样的道理：
![在这里插入图片描述](https://cdn.jsdelivr.net/gh/fjh1997/CSDN@main/source/images/8023ce2bfa4c156a8c98676d01a02433.png)
这个只要按照这个教程：https://askubuntu.com/questions/976391/rename-custom-deb-package把python-dbus.deb    python-gobject-2 里面的依赖的python改为python2.7之后安装即可。

```bash
sudo apt download python-gobject-2
sudo dpkg -x python-gobject-2_2.28.6.1-1+rebuild_amd64.deb  python-gobject
sudo dpkg -e python-gobject-2_2.28.6.1-1+rebuild_amd64.deb  python-gobject/DEBIAN
cd python-gobject/DEBIAN/
vim control
#修改control
sudo dpkg -b python-gobject/  python-gobject-2.deb
# 打包
sudo dpkg -i python-gobject-2.deb
安装
```
