> cp: cannot overwrite non-directory " /home/ctf/lib' with directory '
> /usr/lib ' cp: cannot overwrite non-directory ' /home/ctf/lib32' with
> directory ' /usr/lib32 '  cp: cannot overwrite non-directory '
> /home/ctf/lib64' with directory ' /usr/lib64 cp: cannot ove rwrite
> non-directory ' /home/ctf/libx32' with directory '/usr/libx3'

![在这里插入图片描述](/images/63fd09519a0be4608d42525b890d98a4.png)
解决方法：
https://github.com/fjh1997/pwn-server
如果是ubuntu 19以上（包括19）的镜像，请将base里的Dockerfile
```bash
RUN cp -R /lib* /home/ctf && \
    cp -R /usr/lib* /home/ctf
```
改为
```bash
RUN cp -R /usr/lib* /home/ctf
```
原因是ubuntu19以上采用了软连接模式。/lib*只是一个软链接
